# Incident Journal – Ransomware Attack Analysis

## Overview
This project documents a cybersecurity incident using an incident handler’s journal approach. It focuses on a ransomware attack that impacted a healthcare organization and disrupted its operations.

The journal provides a structured record of the incident, including key details, analysis, and observations, following standard incident response practices.

---

## Objective
To document and analyze a security incident in a clear and structured format, enabling effective communication with stakeholders and supporting future incident response improvements.

---

## Incident Summary
A ransomware attack targeted a healthcare clinic, encrypting critical patient data and preventing employees from accessing essential systems. The attack originated from a phishing campaign that led to malware installation and unauthorized access to the network.

---

## Key Elements Documented
- Incident description  
- Tools used during investigation and response  
- Timeline and contextual details  
- Analysis using the “5 W’s” methodology (Who, What, When, Where, Why)  
- Observations and lessons learned
