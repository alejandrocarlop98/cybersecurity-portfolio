**Incident handler's journal**

| Date:  Tuesday, September 9th 2025 | Entry: \#20250909-001 |  |  |
| :---- | :---- | ----- | ----- |
| Description | Ransomware attack at a small U.S. healthcare clinic that disrupted operations and encrypted critical patient files. |  |  |
| Tool(s) used | Antivirus Software, SIEM monitoring, email filtering system. |  |  |
| The 5 W's  | **Who** caused the incident? An organized group of cybercriminals known for targeting healthcare and transportation sectors. **What** happened? A phishing campaign led to malware installation and subsequent ransomware deployment, encrypting essential medical files. **When** did the incident occur? Tuesday morning, around 09:00 a.m. **Where** did the incident happen? Within the healthcare clinic’s internal IT network and computer systems. **Why** did the incident happen? Employees were deceived into downloading malicious email attachments, which gave attackers access to the system. |  |  |
| Additional notes | The incident highlights the critical need for employee cybersecurity awareness training and stronger email filtering controls. It also raises important questions about the organization’s preparedness and response strategy: *Were reliable data backups available and tested to ensure business continuity without paying the ransom?* and *What additional network monitoring or endpoint detection measures could have identified the phishing attack earlier?* |  |  |
